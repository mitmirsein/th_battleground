# 🌿 에덴 동산지기 (Eden Gardener)
## 스토리텔링 & 제품 기획 문서

> **"첫 아담은 실패했습니다. 당신은 다를 수 있습니다."**

---

## 1. 서사적 기반 (Narrative Foundation)

### 1.1 원형 이야기 (The Original Story)

창세기 2장, 하나님은 에덴동산을 창조하시고 사람에게 맡기셨습니다.

> "여호와 하나님이 그 사람을 이끌어 에덴동산에 두어 그것을 **경작하며(עָבַד, 아바드)** **지키게(שָׁמַר, 샤마르)** 하시고" (창 2:15)

아담(אָדָם)과 이브(חַוָּה)는 **최초의 동산지기**였습니다.

하지만 그들은 동산을 끝까지 지키지 못했습니다.

뱀의 유혹, 선악과, 그리고 추방.

에덴의 문은 닫혔고, 생명나무로 가는 길은 막혔습니다.

### 1.2 미완의 서사 (The Unfinished Story)

성경의 마지막 책 요한계시록은 에덴을 다시 보여줍니다.

> "강 좌우에 **생명나무**가 있어 열두 가지 열매를 맺되 달마다 그 열매를 맺고 그 나무 잎사귀들은 만국을 **치료**하기 위하여 있더라" (계 22:2)

에덴은 잃어버린 과거가 아니라, **회복될 미래**입니다.

### 1.3 당신의 이야기 (Your Story)

**에덴 동산지기**는 이 회복의 여정에 당신을 초대합니다.

- 매일 하나님의 말씀으로 동산에 **물**을 주고
- 매일 기도로 동산의 생명들에게 **양식**을 주며
- **365일** 동안 동산을 지킵니다

첫 아담과 이브가 실패했던 그 여정을, 당신은 완주할 수 있습니다.

## 2. 핵심 콘셉트 (Core Concept)

### 2.1 한 문장 정의

> **"365일 동안 나만의 에덴동산을 가꾸는 기도 일기 웹앱"**

### 2.2 핵심 경험

| 요소 | 설명 |
|:---|:---|
| **정체성** | 사용자는 아담 또는 이브가 되어 동산지기 역할을 맡습니다 |
| **돌봄** | 매일 기도를 기록하면 식물에 물을, 동물에 먹이를 줍니다 |
| **성장** | 생명체들이 시각적으로 자라며, 새로운 생명체가 해금됩니다 |
| **위기** | 기록을 쉬면 생명체들이 시들지만, 언제든 회복 가능합니다 |
| **완주** | 365일 완주 시 "생명나무"가 해금되며 에덴이 회복됩니다 |

### 2.3 감정 곡선

```
열정                    ✨생명나무✨
  │                         ╱
  │    🌳100일           ╱
  │      ╱             ╱
  │    ╱    (슬럼프)  ╱
  │  ╱        ╲    ╱
  │╱           ╲╱
  └─────────────────────────────────────▶ 일수
  Day 1        Day 50      Day 200     Day 365
```

## 4. 케어 시스템 (Care System)

### 4.1 이원화된 돌봄

| 대상 | 액션 | 트리거 | 의미 |
|:---|:---|:---|:---|
| 🌿 **식물** | 💧 물 주기 | 말씀 읽기 | 말씀이 마음을 적신다 |
| 🐑 **동물** | 🍞 먹이 주기 | 기도 쓰기 | 기도가 영혼을 먹인다 |

### 4.2 생명 상태 (Life States)

```
     [씨앗/알]  ──▶  [새싹/부화]  ──▶  [자람]  ──▶  [번성]  ──▶  ✨[황금]✨
       Day 1          Day 3          Day 7        Day 28        Day 100
                                                    
                                    ⬇️ (2일 미돌봄)
                                    
                                 [시들음/배고픔]
                                    │
                                    ▼
                              (돌봄 시 회복)
```

### 4.3 시들음의 신학 (Theology of Withering)

**시들음은 죽음이 아닙니다.**

> "의인은 일곱 번 넘어져도 다시 일어나려니와" (잠 24:16)

- 모든 생명체는 **회복 가능**합니다
- 시들음 상태에서도 완전히 죽지 않습니다
- 다시 돌보면 이전 상태로 돌아갑니다
- **은혜의 메시지**: "괜찮아요, 다시 시작할 수 있어요"

## 6. 생명체 목록 (Creature Catalog)

### 6.0 해금 일정 설계 원칙

- 🌿 **식물**과 🐑 **동물**은 **같은 날 해금되지 않음**
- 평균 **10~15일 간격**으로 새로운 생명체 해금
- 초반에 자주, 후반에 간격 넓히며 **기대감 유지**
- 365일 완주 시 **생명나무 + 어린양&사자** 동시 해금 (대미)

## 7. 기술 명세 (Technical Spec)

### 7.1 플랫폼 전략: PWA (Progressive Web App)

**왜 PWA인가?**

| 고려사항 | 네이티브 앱 | PWA (선택) |
|:---|:---|:---|
| 개발 비용 | iOS + Android 별도 | 단일 코드베이스 ✅ |
| 앱스토어 심사 | 필요 (1~2주) | 불필요 ✅ |
| 앱스토어 수수료 | 15~30% | 없음 ✅ |
| 설치 용량 | 50~100MB | 5~10MB ✅ |
| 업데이트 | 스토어 재심사 | 즉시 배포 ✅ |
| 오프라인 지원 | ✅ | ✅ (Service Worker) |
| 푸시 알림 | ✅ | ⚠️ iOS 제한있음 |

### 7.2 플랫폼별 지원 현황

#### 📱 iOS (iPhone/iPad)

| 기능 | 지원 | 비고 |
|:---|:---|:---|
| 홈 화면 추가 | ✅ | Safari → 공유 → 홈 화면에 추가 |
| 오프라인 사용 | ✅ | Service Worker 지원 (iOS 11.3+) |
| 푸시 알림 | ⚠️ | iOS 16.4+ 필요, 홈 화면 설치 후만 가능 |
| 전체 화면 | ✅ | `display: standalone` |
| 로컬 저장 | ⚠️ | 7일 미사용 시 데이터 삭제 위험 → IndexedDB 사용 |

**iOS 대응 전략:**
1. **푸시 대안**: 이메일 리마인더 옵션 제공
2. **데이터 보호**: IndexedDB + 수동 백업(JSON Export) 기능
3. **설치 가이드**: 첫 방문 시 "홈 화면에 추가" 안내 모달

#### 🤖 Android

| 푸시 알림 | ✅ | 완벽 지원 (Firebase 불필요) |

**Android 장점:**

## 9. 핵심 질문에 대한 답

### Q: 왜 365개가 아니라 24개인가요?

**A**: 365개의 일회성 스탬프보다, 24개의 **살아있는 생명체**와 관계를 맺는 것이 더 깊은 케어 본능을 자극합니다. "오늘 내 포도나무가 목말라해"라는 감정은 "오늘 스탬프 #47을 찍어야지"보다 강력합니다.

### Q: 실패하면 어떻게 되나요?

**A**: **시들지만 죽지 않습니다.** 첫 아담은 에덴을 떠났지만, 당신의 생명체들은 기다립니다. 이것이 은혜의 복음입니다. 돌아오면 회복됩니다.

### Q: 365일 완주하면 끝인가요?

**A**: **생명나무가 해금됩니다.** 그리고 새로운 해가 시작됩니다. 에덴은 한 번 회복되면 영원히 지속됩니다. 2년차, 3년차 동산지기가 되는 것입니다.
